This page records the current LinxISA v0.57 architecture contract. The v0.57

## Updated Block Definition

LinxISA v0.57 defines a block as a block-start header followed by block
descriptors and block body/termination records. Tile-capable blocks use the
descriptor families below as the current architectural contract:

| Descriptor family | Role |
| --- | --- |
| `BSTART.*` | Selects the block class, operation family, and primary execution attributes. |
| `B.DATR` | Carries data attributes such as type extension, pad value, round mode, saturation, compare mode, and byte id. |
| `B.ARG` | Carries layout, format, transform, mode, and profile-specific block arguments. |
| `B.ITP` | Binds input TileReg operands in source pairs. |
| `B.OTA` | Binds output TileReg operands and allocates output tile capacity. |
| `B.META` | Binds an ordinary GPR metadata operand for element-domain shape, valid-region, and mask state. |
| `B.DIM` | Carries matrix/CUBE dimensions where the opcode profile requires M/N/K descriptors. |
| `B.IOR` | Binds ordinary scalar and memory operands. |
| `B.IOD` | Binds dependency operands and dependency-domain records. |
| `B.TEXT` | Selects an out-of-line block body when the block class uses body text. |
| `BSTOP` | Terminates the block record. |

## v0.57 Architectural Updates

1. TileReg operands in `B.ITP` and `B.OTA` are 8-bit architectural ids.
2. TileReg id `0x00` is `TZERO`. It is source-only and is illegal as a
   destination.
3. `B.ITP` binds two source TileReg ids plus per-source reuse hints.
4. `B.OTA` binds one destination TileReg id and encodes destination allocation
   capacity with `CellCountM1`.
5. `B.OTA.CellCountM1` uses 128-byte CELL units:

   ```text
   CELL_count = CellCountM1 + 1
   TileBytes  = CELL_count * 128
   ```

6. `B.META` names an ordinary GPR source. That GPR carries element-domain
   shape, valid-region, and mask metadata for tile execution.
7. Source TileReg size is read from TileReg allocation metadata and is not
   encoded by `B.ITP`.
8. Tile shape metadata is carried by `B.META` for TEPL/TMA element-domain
   operations.
9. Matrix/CUBE operations use `B.DIM` for M/N/K dimensions when required by the
   opcode profile.

## Normative Rules

1. `B.ITP`, `B.OTA`, and `B.META` are block descriptors, not standalone compute
   instructions.
2. A source slot without an operand MUST encode `TZERO`.
3. Any reuse bit attached to a `TZERO` source MUST be zero.
4. `B.OTA` MUST NOT encode `TZERO` as `DstTile`.
5. `B.ITP.src_pair` and `B.OTA.dst_slot` MUST match the operand ordinals
   defined by the opcode profile.
6. `B.META.MetaGpr` participates in ordinary scalar GPR rename, read, and RAW
   dependency handling.
7. `B.META` MUST NOT carry dtype, layout, pad, round, saturation, byte id,
   memory base/stride, scalar operands, quantization operands, or dependency
   tokens.
8. Reserved descriptor bits MUST be zero.
