# LinxISA v0.57 Downstream Migration Delta

This page records the implementation delta required after the v0.57 PTO block
encoding map. It is intentionally scoped to changed downstream behavior.

## Migration Rule

The v0.57 PTO encoding is not complete downstream until:

- SuperNPUBench and TileOP paths emit v0.57 block descriptors and opcode ids.
- LLVM accepts, prints, lowers, and emits v0.57 PTO block forms.
- QEMU decodes and executes v0.57 block descriptors and opcode ids.
- tools/model recognizes v0.57 FIXP/TSTORE starts and regenerated descriptor rows.
- GFSIM decodes and executes the same v0.57 block descriptors and opcode ids.
- `tools/isa/check_pto_v057_downstream.py --strict` reports `OK`.

## Required Delta

| Area | Required delta |
| --- | --- |
| SuperNPUBench inline tile asm | Replace `B.IOT` with `B.ITP` source descriptors and `B.OTA` destination descriptors. Replace old hardcoded TMA/TEPL ids with the ids in `isa/v0.57/state/pto_encoding.json`. |
| TileOP Linx convenience API | Linx TEPL constants now match v0.57 ids and are checker-validated. `TCONCAT`/`TCONCATIDX` public wrappers, CPU simulation semantics, and `__LINXISA__` implementation hooks are present. Remaining TileOP work is replacing the temporary `__LINXISA__` concat hook with real FIXP lowering after the compiler FIXP intrinsic lands. |
| LLVM Linx backend | MC/parser/printer/lowering support is present for `B.ITP`, `B.OTA`, v0.57 TMA/CUBE/TEPL ids, FIXP aliases including `TCONCAT` and `TFILLPAD`, 8-bit block-local selectors, and merged form bits. Blockify emits split source/destination descriptors, and old `B.IOT`/`B.IOTI` LLVM target forms are removed. |
| QEMU Linx target | `BSTART.TMA`, `BSTART.CUBE`, `BSTART.FIXP`, and `BSTART.TEPL` begin direct v0.57 tile headers. TMA dispatch uses `MGATHER=0`, `MSCATTER=1`, `TLOAD=2`, `TPREFETCH=3`, and merged `TSTORE/TSTORE_FP=4`. CUBE dispatch uses merged `TGEMV=0` and `TMATMUL=1`. QEMU decodes `B.ITP`/`B.OTA` directly, stores split descriptor state, consumes `B.OTA` CellCountM1 for destination allocation, and obtains TSTORE source sizes from TileReg allocation metadata. |
| tools/model ISA executor | `generated_tables.cpp` exposes `B.ITP` and `B.OTA` descriptor rows instead of `B.IOT`/`B.IOTI`, and its BSTART table surface uses merged v0.57 `BSTART.TMA`, `BSTART.CUBE`, and `BSTART.FIXP` Function/DataType forms. |
| GFSIM BlockISA model | `BSTART` selector decode uses v0.57 TMA/CUBE/TEPL/FIXP ids, TEPL dispatch reads the direct `TEPL.TileOpcode` byte, FIXP instruction names include `TCONCAT` and `TFILLPAD`, FIXP `TMOV` bridges to the existing move path, and the old block-argument size calculator for combined tile descriptors is removed. |

The downstream checker reports two classes of unfinished work when migration
regresses:

- stale patterns: old constructs that must disappear, such as `B.IOT`, old
  TMA/TEPL ids, 5-bit function masks, and unmerged CUBE suffix opcodes.
- missing required patterns: real TileOP API forms and implementation hooks that
  must exist downstream.
