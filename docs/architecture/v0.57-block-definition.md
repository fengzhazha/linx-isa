# LinxISA v0.57 Block Definition

This page defines the current LinxISA v0.57 block descriptor contract.

## Block Structure

A v0.57 block is an ordered record:

1. `BSTART.*`
2. Zero or more block descriptors
3. Optional body selector or body records when required by the block class
4. `BSTOP`

The descriptor stream is decoded as part of the block. Descriptor fields set
operand bindings, metadata, dimensions, attributes, and execution parameters for
the block operation selected by `BSTART.*`.

## TileReg Namespace

Tile operands encoded by v0.57 tile descriptors use an 8-bit architectural
TileReg id.

Assembly uses direct TileReg references `T0` through `T255` for these encoded
ids. `T0` is the direct spelling of id `0x00` and has the `TZERO` source-only
semantics below.

| Encoding | Meaning |
| ---: | --- |
| `0x00` | `T0` / `TZERO`, a zero/null tile source and source-slot padding value. |
| `0x01..0xff` | Ordinary architectural TileReg id, spelled `T1` through `T255`. |

Rules:

- `T0` / `TZERO` MAY appear as a `B.ITP` source.
- `T0` / `TZERO` MUST NOT appear as a `B.OTA` destination.
- Any reuse bit associated with `T0` / `TZERO` MUST be zero.
- Single-input and odd-input-count operations use `T0` / `TZERO` for unused source
  slots.

## B.ITP: Input Tile Pair

`B.ITP` binds two source TileReg operands and their reuse hints.

```asm
B.ITP [Tsrc0<.reuse>, Tsrc1<.reuse>], <last>, src_pair
```

| Bits | Field | Width | Rule |
| --- | --- | ---: | --- |
| `[31:24]` | `SrcTile1` | 8 | Second source TileReg id. |
| `[23:16]` | `SrcTile0` | 8 | First source TileReg id. |
| `[15]` | `L` | 1 | Last tile descriptor in the block. |
| `[14:12]` | `func` | 3 | Fixed `3'b010`. |
| `[11:10]` | `src_pair` | 2 | `00`: src0/1, `01`: src2/3, `10`: src4/5, `11`: src6/7. |
| `[9]` | `S1R` | 1 | Reuse hint for `SrcTile1`. |
| `[8]` | `S0R` | 1 | Reuse hint for `SrcTile0`. |
| `[7]` | reserved | 1 | Must be zero. |
| `[6:4]` | group | 3 | Fixed `3'b001`. |
| `[3:1]` | group | 3 | Fixed `3'b001`. |
| `[0]` | length | 1 | Fixed `1`. |

Rules:

- `src_pair` selects source operand slots in pairs.
- Unused source slots MUST encode `T0` / `TZERO`.
- Any reuse bit attached to `T0` / `TZERO` MUST be zero.
- If a block has no output tile descriptor, the final `B.ITP` MUST set `L=1`.
- `B.ITP` does not encode source size, dtype, layout, shape, memory operands,
  or dependency tokens.

## B.OTA: Output Tile Allocate

`B.OTA` binds one destination TileReg and its allocation size.

```asm
B.OTA ->Tdst<CellCountM1>, <last>, dst_slot
```

| Bits | Field | Width | Rule |
| --- | --- | ---: | --- |
| `[31:24]` | `DstTile` | 8 | Destination TileReg id. |
| `[23:16]` | `CellCountM1` | 8 | Output size in 128-byte CELL units minus one. |
| `[15]` | `L` | 1 | Last tile descriptor in the block. |
| `[14:12]` | `func` | 3 | Fixed `3'b011`. |
| `[11:10]` | `dst_slot` | 2 | `00`: dst0, `01`: dst1, `10`: dst2, `11`: dst3. |
| `[9:7]` | reserved | 3 | Must be zero. |
| `[6:4]` | group | 3 | Fixed `3'b001`. |
| `[3:1]` | group | 3 | Fixed `3'b001`. |
| `[0]` | length | 1 | Fixed `1`. |

Size calculation:

```text
CELL_count = CellCountM1 + 1
TileBytes  = CELL_count * 128
```

| `CellCountM1` | CELL count | Tile bytes |
| ---: | ---: | ---: |
| `0` | 1 | 128 B |
| `1` | 2 | 256 B |
| `3` | 4 | 512 B |
| `7` | 8 | 1 KB |
| `31` | 32 | 4 KB |
| `255` | 256 | 32 KB |

Rules:

- `DstTile=T0` / `TZERO` is illegal.
- `dst_slot` MUST match the opcode profile output ordinal.
- Same-shape elementwise ops MUST use a `CellCountM1` compatible with all
  source TileReg allocation sizes.
- Convert, reduce, expand, and matrix-like ops define their legal source and
  destination size relationships in their opcode profile.

## B.META: Metadata GPR Operand

`B.META` binds one ordinary GPR source as tile execution metadata.

```asm
B.META MetaGpr, MetaMode
```

| Bits | Field | Width | Rule |
| --- | --- | ---: | --- |
| `[31:20]` | reserved | 12 | Must be zero. |
| `[19:15]` | `MetaGpr` | 5 | Ordinary GPR source id. |
| `[14:12]` | `func` | 3 | Fixed `3'b100`. |
| `[11]` | `MetaMode` | 1 | `0`: `meta_shape`, `1`: `meta_mask`. |
| `[10:7]` | reserved | 4 | Must be zero. |
| `[6:4]` | group | 3 | Fixed `3'b100`. |
| `[3:1]` | group | 3 | Fixed `3'b001`. |
| `[0]` | length | 1 | Fixed `1`. |

`MetaGpr` participates in ordinary scalar GPR rename, read, and RAW dependency
tracking.

### meta_shape Payload

When `MetaMode=0`, `MetaGpr[63:0]` is interpreted as:

| Bits | Field | Width | Rule |
| --- | --- | ---: | --- |
| `[3:0]` | `format` | 4 | Format id compatible with the block primary dtype. |
| `[7:4]` | `logR` | 4 | Physical rows are `1 << logR`. |
| `[11:8]` | `logC` | 4 | Physical columns are `1 << logC`. |
| `[13:12]` | `vmode` | 2 | `0`: FULL, `1`: RECT, `2`: TILE_MASK, `3`: reserved. |
| `[27:14]` | `valid.R` | 14 | Valid rows. |
| `[41:28]` | `valid.C` | 14 | Valid columns. |
| `[45:42]` | `off.R` | 4 | Valid rectangle row offset. |
| `[49:46]` | `off.C` | 4 | Valid rectangle column offset. |
| `[57:50]` | `mask_tile` | 8 | Mask TileReg id used when `vmode=TILE_MASK`. |
| `[63:58]` | reserved | 6 | Must be zero. |

Rules:

- `MetaMode=meta_shape` describes element-domain shape, valid rectangle, and
  optional mask tile id.
- `MetaMode=meta_mask` is reserved unless an opcode profile explicitly defines
  its payload.
- `B.META` MUST NOT carry dtype, layout, pad, round, saturation, byte id,
  memory base/stride, scalar operands, quantization operands, or dependency
  tokens.

## Matrix Dimensions

Matrix/CUBE opcodes use `B.DIM` for M/N/K dimensions when the opcode profile
requires explicit matrix dimensions. `B.DIM` does not bind TileReg operands.

## Descriptor Ownership

| Descriptor | Owns |
| --- | --- |
| `BSTART.*` | Block class, operation selection, primary execution attributes. |
| `B.DATR` | Data attributes: type extension, pad value, round mode, saturation, compare mode, byte id. |
| `B.ARG` | Layout, format, transform, mode, and profile-specific arguments. |
| `B.ITP` | Tile source operand binding and source reuse hints. |
| `B.OTA` | Tile destination operand binding and destination allocation size. |
| `B.META` | Element-domain shape, valid-region, and mask metadata through a GPR source. |
| `B.DIM` | Matrix/CUBE dimensions when required by the opcode profile. |
| `B.IOR` | Scalar and memory operands. |
| `B.IOD` | Dependency operands and dependency-domain records. |
| `B.TEXT` | Out-of-line block body selection. |
| `BSTOP` | Block termination. |
