This document is the canonical manual for the **block-structured Linx Instruction Set Architecture (Linx ISA)**, version **v0.56.4**.

== Authority and Source of Truth

The live source of truth for this repository is:

* `isa/v0.56/linxisa-v0.56.json` (compiled canonical catalog)
* `isa/v0.56/state/{kernel_execution,rendering_profile,memory_model,system_registers}.json` (normative machine-readable state contracts)
* `isa/generated/codecs/` (generated encode/decode tables derived from the v0.56 catalog)

Where this manual and the machine-readable catalog disagree, the compiled v0.56 catalog and referenced state files MUST be treated as authoritative.

[IMPORTANT]
====
`v0.56` is the only live canonical ISA profile in this repository. Historical pre-v0.56 material is retained for traceability only and must not be used as the active contract.
====

[[preamble-scope]]
== Scope and Maturity

This manual describes the live **v0.56.4 Linx ISA** profile:

* the v0.2 privilege/MMU baseline and block-structured control-flow model,
* structured `MPAR`/`MSEQ` shader kernels selected by `B.TEXT`,
* `p` as the architectural 64-bit EXEC mask with `V.CMP.* ->p` mask generation,
* one unified `lx64` kernel encoding space where `l.*` versus `v.*` is derived from operand class,
* canonical bridged kernel memory through `l.*.brg` and `v.*.brg`,
* the first frozen rendering profile: 4KB `1024x1` row-major SoA fragment/state carriers, TAU tile-to-tile only, and descriptor-vs-StateTile state placement rules.

The v0.56.4 profile remains the only live canonical profile in this repository;
future ISA deltas must be introduced as new explicit contract pages before any
toolchain, emulator, RTL, or manual surface treats them as live behavior.

The LinxISA project maintains alignment between:

* The machine-readable catalog (`isa/v0.56/linxisa-v0.56.json`)
* The LLVM backend (`compiler/llvm/llvm/lib/Target/Linx/`)
* The QEMU emulator (`emulator/qemu/target/linx/`)
* The LinxCore architecture contract (`docs/architecture/linxcore/`)
* This manual

This manual focuses on:

* architectural state and assembly conventions,
* the block-structured control-flow model (`BSTART/BSTOP` and `BARG`),
* template instructions (for example `FENTRY`, `FRET.*`) used by toolchains,
* the canonical shader-kernel and rendering-profile contracts,
* privileged architecture (ACR, SSR, trap envelope), and
* a complete instruction reference generated from the v0.56 catalog.
